# Research: Competitive & Inspiration Landscape

**Date:** 2026-07-15

## What exists

| Product / class | Notes | Gap vs Eye Scroll |
|-----------------|-------|-------------------|
| Android Switch Access | Official; external switches | Not camera wink-based |
| Voice Access | Voice commands | Hands-free but not silent |
| Face Control / FaceJoy-style apps | Some face → tap | Rarely Instagram-optimized swipe + always-on |
| Gamified blink trainers | Detection only | No system gestures |
| Wear OS / glasses gaze | Hardware | Not phone front-cam |

## Differentiation for Eye Scroll

1. **Feed-native core set** — next/prev = vertical swipes matching Reels/Shorts  
2. **Volume via both-then-one-eye** — low accidental change while watching  
3. **Nose expand = tap** — creative, distinguishable from blinks  
4. **MIUI survival UX** as first-class onboarding  
5. **Toggleable extras** without cluttering core reliability  
6. **Open APK CI** via GitHub Actions for sideload iteration  

## Risks

- False winks from laughing / rubbing eyes → cooldown + calibration  
- Platform policy if publishing on Play (Accessibility + background camera) — sideload first  
- Battery heat on long Reels sessions → FPS adaptive + Pause in notification
